Performance Review — Руководство по сборке и запуску проекта
Общее описание
Performance Review — веб‑приложение для управления целями сотрудников, сбора обратной связи и анализа результатов оценок.

Основные функции:

управление целями и ключевыми задачами сотрудников (создание, редактирование, удаление);

система напоминаний о приближающихся сроках;

сбор и просмотр обратной связи (самооценка, оценки коллег и руководителя);

аналитика: расчёт баллов, формирование рейтингов, генерация рекомендаций;

дашборд с прогрессом целей команды;

тёмная тема интерфейса по умолчанию.

Проект использует тестовые данные. Возможна интеграция с корпоративным порталом через API.

Стек технологий
Фронтенд
React — библиотека для построения интерфейсов;

Ant Design — UI‑компоненты и базовые стили;

Recharts — визуализация данных и графики;

CSS Modules — локальные стили компонентов;

UUID — генерация уникальных идентификаторов;

Fetch API — взаимодействие с бэкендом.

Бэкенд
Node.js / Express — серверная часть;

Prisma — ORM для работы с SQLite;

JWT — аутентификация и авторизация;

dotenv — управление переменными окружения;

CORS — настройка доступа с фронтенда.

Примечания:

Reset.css не используется (базовые стили предоставляет Ant Design);

интерфейс выполнен в тёмной теме с плавными анимациями.

Инструкция по сборке и запуску
1. Запуск бэкенда
Перейдите в директорию бэкенда:

bash
cd backend
Установите зависимости:

bash
npm install
Задайте переменные окружения:

bash
DATABASE_URL="file:./dev.db"
JWT_SECRET="ваш_секрет"
PORT=4000
Запустите сервер:

bash
npm run dev
Результат: сервер будет доступен по адресу http://localhost:4000.

2. Запуск фронтенда
Перейдите в директорию фронтенда:

bash
cd frontend
Установите зависимости:

bash
npm install
Запустите приложение:

bash
npm start
Результат: приложение откроется в браузере по адресу http://localhost:5173.

Примечания по настройке
Для работы бэкенда требуется Node.js (рекомендуется последняя стабильная версия).

База данных SQLite будет создана автоматически в директории бэкенда (dev.db).

Значение JWT_SECRET следует заменить на уникальный секретный ключ для вашей среды.

Порт 4000 (бэкенд) и 5173 (фронтенд) можно изменить в переменных окружения и конфигурационных файлах соответственно.
